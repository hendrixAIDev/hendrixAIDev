# Content / Launch

**Purpose:** Guide CTO planning for content, messaging, publishing, and launch work.

**What this covers:**
- When to use the content/launch workflow
- Planning expectations before dispatch
- Typical content production, review, validation, and closure path

Use when the output is content, messaging, publishing, or launch material rather than product code.

Default path:
1. CTO triage and planning
2. Content dispatch
3. Review or validation as needed
4. CTO acceptance decision
5. Close

CTO planning should define:
- target audience and channel
- source material the content worker must use
- approval, legal, trust, or brand constraints
- what evidence proves the deliverable is ready

Notes:
- use for launch posts, articles, product messaging, and similar work
- validation should match the deliverable, not default to engineering QA
