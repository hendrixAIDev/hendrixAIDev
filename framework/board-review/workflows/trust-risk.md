# Trust / Risk / Policy

**Purpose:** Guide CTO planning for legal, trust, safety, or policy-sensitive tickets.

**What this covers:**
- When to use the trust/risk workflow
- Planning expectations before dispatch
- Typical review, escalation, follow-up, and closure path

Use when legal, trust, safety, or policy concerns dominate the ticket.

Default path:
1. CTO triage and planning
2. Trust / risk review
3. CTO decision
4. Escalate to JJ if needed, otherwise close or create follow-up work

CTO planning should define:
- the specific risk or policy question
- what evidence the reviewer must inspect
- which decisions are within CTO authority
- what would require JJ escalation

Notes:
- use `status:needs-jj` only when a real CEO decision is required
- validation is policy/risk review, not product QA by default
