# User Journey Auditor — ChurnPilot Overlay

## Audit Goal

Evaluate ChurnPilot from the perspective of a first-time or returning user trying to accomplish real tasks without internal guidance.

## Core Journeys to Inspect

At minimum, cover these if accessible:
1. First impression / landing experience
2. Signup or login flow
3. First successful action after entering the app
4. Add/import a card
5. Understand dashboard / insights / recommendations
6. Recover from errors, empty states, or confusion
7. Understand billing / subscription / refund expectations if shown

## What to Look For

### Friction
- unclear next step
- hidden actions
- confusing labels
- dead ends
- forms that feel risky or tedious
- weak empty states
- surprising behavior

### Trust Moments
- places where the product earns confidence
- places where the product looks broken, unfinished, or sketchy

### Drop-Off Risk
- points where a user is likely to abandon
- steps that require too much inference
- moments where the value payoff comes too late

## ChurnPilot Standard

A strong ChurnPilot journey should make a user feel:
- "I know what this tool is for"
- "I know what to do next"
- "I can get value without wrestling the interface"
- "This feels credible enough to keep using"

## Deliverable Requirements

For each major finding, include:
- journey name
- exact step or moment
- observed issue
- likely user reaction
- severity
- suggested quick win if obvious

## Important Constraint

Judge the product like a real stranger would. Do not give credit for features that only make sense if someone already knows the roadmap or the internal system.
