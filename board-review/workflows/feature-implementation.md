# Feature Implementation

**Purpose:** Guide CTO planning for tickets that build or change product behavior.

**What this covers:**
- When to use the feature implementation workflow
- Planning expectations before dispatch
- Typical implementation, review, validation, and closure path

Use when the ticket requires building or changing product behavior.

Default path:
1. CTO triage and planning
2. Engineer implementation
3. Code review
4. Validation
5. CTO acceptance decision
6. Close

CTO planning should define:
- the user-facing outcome
- acceptance tests or acceptance criteria
- implementation ownership and role
- review expectations
- validation mode before closure

Notes:
- default for most product code tickets
- requires an explicit validation plan
- use acceptance tests as the implementation target
