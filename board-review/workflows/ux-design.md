# UX / Design

**Purpose:** Guide CTO planning when interaction design, user journey, or layout is unclear.

**What this covers:**
- When to use the UX/design workflow
- Planning expectations before dispatch
- Typical design exploration, CTO decision, and follow-up path

Use when interaction design, user journey, or layout needs to be clarified before implementation.

Default path:
1. CTO triage and planning
2. UX / design dispatch
3. CTO decision
4. Create implementation tickets or close

CTO planning should define:
- the user journey or interaction problem
- product constraints and existing design conventions
- what deliverable is needed: recommendation, flow, wireframe, copy, or follow-up ticket
- what evidence would make the design ready for implementation

Notes:
- use before engineering when the product shape is still unclear
- output may be recommendations, flows, or follow-up tickets
